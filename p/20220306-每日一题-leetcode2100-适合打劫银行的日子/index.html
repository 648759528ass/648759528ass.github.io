2100. 适合打劫银行的日子

题面

你和一群强盗准备打劫银行。给你一个下标从 0 开始的整数数组 security ，其中 security[i] 是第 i 天执勤警卫的数量。日子从 0 开始编号。同时给你一个整数 time 。

如果第 i 天满足以下所有条件，我们称它为一个适合打劫银行的日子：

第 i 天前和后都分别至少有 time 天。 第 i 天前连续 time 天警卫数目都是非递增的。 第 i 天后连续 time 天警卫数目都是非递减的。 更正式的，第 i 天是一个合适打劫银行的日子当且仅当：security[i - time] >= security[i - time + 1] >= … >= security[i] <= … <= security[i + time - 1] <= security[i + time].

请你返回一个数组，包含 所有 适合打劫银行的日子（下标从 0 开始）。返回的日子可以 任意 顺序排列。

解析

这道题一开始没想到前缀和的解法，往滑动窗口那边去想了，所以这道题解法其实挺多样的。

双指针滑动窗口

针对左边界l与右边界r。若其距离里面的日子数量为 2*time + 1 时，该段为一个合适的时间段，其中中间即 (l+r)/2 这天即为合适的日子。

那么我们可以令r向前找，然后想办法迁移l。 显然在

在$r\le l+time$时，必有 $ security[r - 1] \ge security[r]$, 若$ security[r - 1] \lt security[r] $ 则 $l$ 迁移到 $r+1$ 处

在 $ r\ge l+time $时, 必有 $ security[r - 1] \le security[r]$, 若$security[r-1] \gt security[r]$ 则 $l$ 迁移到 $r+1$

当 $ r - l == 2*time $ 时， 此时$(r+l)/2$ 即为合适的日子，同时如果$security[(r+l)/2] == security[(r+l)/2 + 1]$ 则将l++。

如果不等的话,将l迁移到当前$security[r] $ 值的最前处

比如

[1,1,1,2,2,0,1,1]

在 $l = 0, r = 4 $ 处会获得一个值，则我们需要把$l$ 迁移到 3 处，为了完成这一步骤，我们可以用一个标记来特殊标记r的连续数.

前缀和

前缀和就没有那么多特殊讨论了，只需要先做一遍递减前缀数量，然后做一遍递增后缀数量即可。当前缀数量和后缀数量大于time时，当前位置就是对应的数量

时间复杂度:$O(n)$

class Solution {
public:
    vector<int> goodDaysToRobBank(vector<int>& security, int time) {
        vector<int> pre(security.size(),0),r_pre(security.size(),0),ret;
        for(int i = 1;i<security.size();i++){
            if(security[i-1]>=security[i]){
                pre[i] = pre[i-1]+1;
            }
            
        }
        for(int i = security.size()-2;i>=0;i--){
            if(security[i]<=security[i+1]){
                r_pre[i] = r_pre[i+1]+1;
            }
        }
        for(int i = 0;i<security.size();i++){
            if(pre[i]>=time&&r_pre[i]>=time){
                ret.push_back(i);
            }
        }
        return ret;
    }
};

class Solution {
public:
    vector<int> goodDaysToRobBank(vector<int>& security, int time) {
        vector<int> ret;
        int l = 0,r =0,mov = 0;
        while(r<security.size()){
            if(r-l == time*2){
                int idx = (r+l)>>1;
                ret.push_back(idx);
                if(idx+1< security.size() && security[idx] == security[idx+1]){
                    l++;
                }else if(time == 0){
                    l++;
                }else{
                    l = mov;
                }
            }
            if(++r >= security.size()) break;
            if(r<=l+time){
                if(security[r]>security[r-1]){
                    l = r;
                }
            }else if(r == l+time + 1){
                if(security[r]<security[r-1]){
                    l++;
                }
                mov = r;
            }else{
                if(security[r]<security[r-1]){
                    l = mov;
                }else if(security[r]>security[r-1]){
                    mov = r;
                }
            }
            
        }
        return ret;
    }
};
