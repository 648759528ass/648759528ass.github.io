798. 得分最高的最小轮调

题面

给定一个数组 $A$，我们可以将它按一个非负整数 $K$ 进行轮调，这样可以使数组变为 $ A[K], A[K+1], A[K+2], … A[A.length - 1], A[0], A[1], …, A[K-1] $ 的形式。此后，任何值小于或等于其索引的项都可以记作一分。 例如，如果数组为 $[2, 4, 1, 3, 0]$，我们按 K = 2 进行轮调后，它将变成 $[1, 3, 0, 2, 4]$。这将记作 3 分，因为 1 > 0 [no points], 3 > 1 [no points], 0 <= 2 [one point], 2 <= 3 [one point], 4 <= 4 [one point]。

在所有可能的轮调中，返回我们所能得到的最高分数对应的轮调索引 $K$。如果有多个答案，返回满足条件的最小的索引 $K$。

解析

这道题非常容易想到暴力解法，但是正确思路跟暴力解法是完全不一样的。

暴力解法:将数组二倍展开，然后依题意模拟计算每一个k的分数

差分解法：

对于差分解法我们首先需要推导一下结论，这点我觉得是没有经验的人比较难以想到的。

思路不再从k开始，我们首先从每一个A[i]开始

对于每一个A[i] , 假若在k处进行轮调， 则论调后的下标为 $(i-k+n)%n$ ,我们可以讨论k

对于 $k < i$ , 则其轮调后下标为 $i-k$;

对于$ k > i$,则其轮调后的下标为 $i - k+n$

我们再来考虑 A[i]的值，

若$A[i] \le i$; 在 k<= i 的情况下 , k 每加 1 对应下标就减去1， 若使得$A[i]\le i-k$ 即$k\le A[i] - i$ ；

在$k>i$ 的情况，显然 有 $i-k+n > i \ge A[i]$ 即$k>i$的数完全成立

若$A[i] \gt i$; 在 k<= i 的情况下 , k 每加 1 对应下标就减去1，则完全不能有k满足 $A[i] \le i-k$

在$k>i$的情况，有 $i-k+n > i$ ，如果我们要使得 $ i-k+n>A[i] $ 即我们需要$k\lt i+n - A[i]$ ，则我们可以得出结论 $i<k<i+n-A[n]$

对于上述情况，每一个i能使得加分数的k都是连续的一段，则我们只需要差分记录开始点和结束点，即可在$O(n)$时间复杂度内推算出所有k

总的来说，暴力的思路是 k -> i

而差分的思路是 i->k

错误解法 暴力

class Solution {
public:
    int bestRotation(vector<int>& nums) {
        int n = nums.size();
        for(int i = 0;i<n;i++){
            nums.push_back(nums[i]);
        }
        pair<int,int> ans = {0,0};
        for(int i = 0;i<n;i++){
            int cnt = 0;
            for(int j = 0;j<n;j++){
                if(nums[i+j] <= j)cnt++;
            }
            if(cnt>ans.second){
                ans = {i,cnt};
            }
        }
        return ans.first;
    }
};

class Solution {
public:
    int bestRotation(vector<int>& nums) {
        int n = nums.size();
        vector<int> pre(n+1,0);
        for(int i = 0;i<nums.size();i++){
            if(i<nums[i]){
                pre[i+1] += 1;
                pre[i-nums[i]+n+1] -= 1;
            }else{
                pre[i+1] += 1;
                pre[0] += 1;
                pre[i - nums[i] + 1] -= 1;
            }
        }
        int ans = 0;
        for(int i = 1;i<n;i++){
            pre[i]+=pre[i-1];
            if(pre[i] > pre[ans]){
                ans = i;
            }
        }
        return ans;
    }
};
